Experiment 01 — Multi-Agent System

Agentic Personal Assistant

A personal AI that answers questions about me by orchestrating five specialist sub-agents, each reading from a different data source — SQLite, live portfolio scrape, semantic vector store, Google Drive, and Gmail — and synthesising results into a single grounded response.

The orchestrator never answers from its own training knowledge. It always calls tools first, gets real data, then generates a response. Every answer cites its source.

Stack Python, Anthropic Claude, ChromaDB, FastAPI, Gradio
Deployed Hugging Face Spaces (Docker)

What I built & understand

01

Multi-agent orchestration

A central LLM routes questions to specialist agents based on tool descriptions alone — no rules or hard-coded logic.

02

Tool-calling loops

The orchestrator loops: call LLM → tool requests → run tools → inject results → repeat until stop_reason is end_turn. Implemented for both Anthropic and OpenAI formats.

03

RAG pipelines

Documents are chunked, embedded into vectors with OpenAI, stored in ChromaDB. At query time the question is embedded and compared by cosine similarity to retrieve grounding context.

04

Parallel execution

ThreadPoolExecutor runs multiple agents simultaneously. A 1s SQLite call and a 3s web scrape finish at second 3, not second 4.

05

Async streaming

Final answers stream token-by-token via AsyncAnthropic. Status events ("Searching knowledge base…") keep the UI responsive during tool-call rounds.

06

Provider-agnostic design

Swapping from Claude to GPT-4o or a local Ollama model is one line in config.py. Tool schemas auto-convert between Anthropic and OpenAI formats.
